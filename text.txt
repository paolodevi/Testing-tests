Points 5 (animation) and 6 (shape-density map) are particularly useful when the shape itself matters, not just a scalar quantity like area.

5. Animation of Patch Evolution

The goal is to understand how the boundary deforms as fwdF and ctrF change.

There are several ways to do this.

Option A: Sweep one parameter, hold the other fixed

Suppose:

fwdF = [0.8 1.0 1.2 1.4];
ctrF = [0.5 1.0 1.5 2.0];

and your patches are stored as

x{i,j}
y{i,j}

where:

i -> fwdF
j -> ctrF

Choose a fixed ctrF index:

j0 = 2;

figure

for i = 1:length(fwdF)

    clf

    patch(x{i,j0},y{i,j0}, ...
          'b', ...
          'FaceAlpha',0.3, ...
          'LineWidth',2)

    axis equal
    grid on

    title(sprintf( ...
        'fwdF = %.2f   ctrF = %.2f', ...
        fwdF(i),ctrF(j0)))

    drawnow
    pause(0.2)

end

You immediately see whether the patch:

expands,

contracts,

translates,

rotates,

changes topology.



---

Option B: Animate through the full parameter grid

Think of the simulation results as a movie:

figure

for i = 1:length(fwdF)

    for j = 1:length(ctrF)

        clf

        patch(x{i,j},y{i,j}, ...
              'b', ...
              'FaceAlpha',0.3)

        axis equal
        grid on

        title(sprintf( ...
            'fwdF = %.2f   ctrF = %.2f', ...
            fwdF(i),ctrF(j)))

        drawnow

    end

end

This is effectively a walk through parameter space.


---

Option C: Show previous patches as "ghosts"

This is often much more informative.

figure
hold on

for i = 1:length(fwdF)

    patch(x{i,j0},y{i,j0}, ...
        'b', ...
        'FaceAlpha',0.05, ...
        'EdgeColor',[0.6 0.6 0.6]);

end

for i = 1:length(fwdF)

    h = patch(x{i,j0},y{i,j0}, ...
        'r', ...
        'FaceAlpha',0.4);

    drawnow

    pause(0.2)

    delete(h)

end

The grey patches provide context while the red patch moves through the sequence.

This often reveals trends that are difficult to notice in a simple animation.

6. Shape Density / Occupancy Map

This is a very powerful visualization.

Interpretation

Imagine laying all patches on top of one another.

For every point in the plane, ask:

> How many parameter combinations contain this point?



The result is a density map.


---

Example

Suppose you have 100 simulations.

For one location:

(x,y)

contained in 100 patches → density = 100

contained in 50 patches → density = 50

contained in 0 patches → density = 0


Plotting this creates a map showing:

core regions that always exist,

marginal regions that only appear for certain parameter values.



---

Step 1: Build a Common Grid

Determine plotting limits:

xmin = min(cellfun(@min,x(:)));
xmax = max(cellfun(@max,x(:)));

ymin = min(cellfun(@min,y(:)));
ymax = max(cellfun(@max,y(:)));

Create a grid:

Nx = 300;
Ny = 300;

[X,Y] = meshgrid( ...
    linspace(xmin,xmax,Nx), ...
    linspace(ymin,ymax,Ny));


---

Step 2: Accumulate Occupancy

Initialize:

density = zeros(size(X));

Loop through all simulations:

for i = 1:length(fwdF)

    for j = 1:length(ctrF)

        mask = inpolygon( ...
            X,Y, ...
            x{i,j},y{i,j});

        density = density + mask;

    end

end


---

Step 3: Normalize

Convert to percentage:

density = density / ...
          (length(fwdF)*length(ctrF));

Now:

1.0 = always occupied
0.5 = occupied half the time
0.0 = never occupied


---

Step 4: Plot

figure

imagesc( ...
    X(1,:), ...
    Y(:,1), ...
    density)

axis xy
axis equal

xlabel('x')
ylabel('y')

colorbar

The colorbar becomes:

0     Never appears
0.5   Appears in 50% of cases
1.0   Appears in every case


---

Variant: Weight by Sensitivity

Instead of counting every patch equally, weight them by parameter distance from a nominal design.

For example:

w = sqrt( ...
    ((fwdF(i)-fwd_nom)/dfwd).^2 + ...
    ((ctrF(j)-ctr_nom)/dctr).^2 );

Then:

density = density + w*mask;

This highlights which parts of the region are influenced most strongly by parameter changes.


---

A Hybrid Visualization I Would Recommend

For sensitivity studies, I often find the following combination most revealing:

1. Plot the nominal patch in black.


2. Plot the occupancy map as a colored background.


3. Overlay the outer envelope of all patches.



Conceptually:

+-------------------------+
|      envelope           |
|   ##################    |
|   #### nominal ####     |
|   ##################    |
+-------------------------+

where the color intensity indicates how frequently that region occurs.

This immediately shows:

the robust core region,

the sensitive boundary regions,

the overall variation envelope.


For design-space, stability-region, or feasible-region studies, that single figure is often more informative than dozens of individual patch plots.
