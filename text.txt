A good way to approach this in MATLAB is to build a call graph starting from a top-level function.

The key challenges are:

1. Add all source folders to the MATLAB path.


2. Parse each .m file to find function calls.


3. Determine whether each called function is:

a MATLAB built-in,

a toolbox function,

or a user-defined function in your source directories.



4. Recursively process user-defined functions.


5. Avoid infinite loops when functions call each other.



MATLAB provides useful functions such as which() and exist() that can help distinguish built-ins from user code.

Here's a starting implementation:

function callTree = analyzeDependencies(mainFunction, sourcePaths)

    % Add source directories to path
    for k = 1:numel(sourcePaths)
        addpath(genpath(sourcePaths{k}));
    end

    processed = containers.Map();
    callTree = processFunction(mainFunction, processed);

end


function node = processFunction(funcName, processed)

    node.name = funcName;
    node.children = {};

    % Prevent recursion loops
    if isKey(processed, funcName)
        return
    end

    processed(funcName) = true;

    funcFile = which(funcName);

    if isempty(funcFile)
        return
    end

    calls = findFunctionCalls(funcFile);

    for k = 1:length(calls)

        calledFunc = calls{k};

        if isBuiltinOrToolbox(calledFunc)
            continue
        end

        child = processFunction(calledFunc, processed);
        node.children{end+1} = child;

    end

end


function tf = isBuiltinOrToolbox(funcName)

    tf = true;

    location = which(funcName);

    if isempty(location)
        return
    end

    if contains(location,'built-in')
        return
    end

    matlabrootPath = matlabroot;

    if startsWith(location, matlabrootPath)
        return
    end

    tf = false;

end


function calls = findFunctionCalls(mfile)

    txt = fileread(mfile);

    % Remove comments
    txt = regexprep(txt,'%.*','');

    % Find function-like patterns
    tokens = regexp(txt,...
        '(?<![\.\w])([A-Za-z]\w*)\s*\(',...
        'tokens');

    calls = unique(string([tokens{:}]));

    keywords = [
        "if","for","while","switch","case","otherwise",...
        "function","end","try","catch","classdef"
        ];

    calls = setdiff(calls,keywords);

    calls = cellstr(calls);

end

Usage:

paths = {
    'C:\Project\Core'
    'C:\Project\Utilities'
    };

tree = analyzeDependencies('mainFunction', paths);

A more robust approach

The regex approach works surprisingly well for many codebases, but MATLAB already contains dependency-analysis tools that are much more accurate:

deps = matlab.codetools.requiredFilesAndProducts('mainFunction.m');

This returns all dependent files MATLAB can detect.

If your goal is simply:

> Starting from mainFunction, list every non-MATLAB function that is eventually called



then requiredFilesAndProducts() is often the easiest solution:

deps = matlab.codetools.requiredFilesAndProducts('mainFunction.m');

for k = 1:numel(deps)
    disp(deps{k})
end

You can then filter out anything under matlabroot:

userFiles = deps(~startsWith(deps, matlabroot));

This is generally more reliable than parsing source text yourself because it understands:

methods,

packages (+pkg),

class definitions,

local functions,

overloaded functions,

toolbox dependencies.


A few questions before going further:

1. Do you want a flat list of all non-built-in dependencies, or a hierarchical call tree showing who calls whom?


2. Is the code mostly procedural .m files, or does it use MATLAB classes (classdef) heavily?


3. Are the two source folders the complete codebase, or are there additional third-party folders on the MATLAB path?
