Good — now we’re dealing with two separate graphical issues:

1. Some VEAS lines intersect the top boundary (h = h_max) but are not labeled


2. The VEAS curves extend outside the envelope (M > 1)



Both are geometric clipping problems. Let’s fix them cleanly.


---

✅ Problem 1 — Missing Labels at the Top Boundary

Right now you are only checking intersection with:

M = 1   (right border)

But some VEAS lines instead intersect:

h = h_max   (top border)

So for each VEAS line you must check both possible intersections and label whichever occurs first.


---

🔹 Geometric Logic

Each VEAS curve can terminate at:

Right boundary → Mach = 1

Top boundary → h = h_max


We compute both candidates and use the physically valid one.


---

✅ Clean Solution

After computing:

M_line = VEAS(i) ./ (a .* sqrt(rho./rho0));

Add this logic:

M_max = 1;
h_max = max(h_ft);

% --- Intersection with right boundary (Mach = 1) ---
diff_right = M_line - M_max;
idx_right = find(diff_right(1:end-1).*diff_right(2:end) < 0,1);

h_right = NaN;

if ~isempty(idx_right)
    h_right = interp1(diff_right(idx_right:idx_right+1), ...
                      h_ft(idx_right:idx_right+1), 0);
end

% --- Intersection with top boundary (h = h_max) ---
M_top = interp1(h_ft, M_line, h_max);

% --- Decide which boundary is hit first ---
if ~isnan(h_right)
    % Check if right intersection is below top
    if h_right <= h_max
        text(1.01, h_right, sprintf('%d KEAS', VEAS_knots(i)), ...
             'FontWeight','bold', ...
             'VerticalAlignment','middle');
    else
        text(M_top, h_max+1000, sprintf('%d KEAS', VEAS_knots(i)), ...
             'FontWeight','bold', ...
             'HorizontalAlignment','center');
    end
else
    % Only top intersection exists
    text(M_top, h_max+1000, sprintf('%d KEAS', VEAS_knots(i)), ...
         'FontWeight','bold', ...
         'HorizontalAlignment','center');
end


---

✅ Problem 2 — Lines Extending Outside the Envelope

Right now you plot:

plot(M_line, h_ft)

But for altitudes where M_line > 1, it extends outside your Mach limit.

You must clip the curve:

M_line(M_line > 1) = NaN;

So change your plotting loop to:

for i = 1:length(VEAS)

    M_line = VEAS(i) ./ (a .* sqrt(rho./rho0));
    
    % Clip outside envelope
    M_line(M_line > 1) = NaN;
    
    plot(M_line, h_ft, '--')
end

MATLAB automatically breaks the line at NaN.

This is the correct professional way to clip curves.


---

🔎 Why This Fix Works

Now each VEAS curve:

Stops exactly at Mach = 1

Or reaches the top boundary cleanly

Gets labeled at whichever boundary it touches


No overflow. No missing labels.
